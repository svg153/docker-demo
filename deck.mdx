# Docker concepts and interactions

- **Host**, the machine that is running the Docker engine, necessary to run the containers.
- **Layer**, a hierarchy of files, with meta-data.
- **Image**, consists of read-only layers each of which represents a Dockerfile instruction
- **Container**, a contained running process, started from an image.
- **Registry**, a repository of images. Local or remote.
- **Volume**, storage outside the container.
- **Dockerfile** (`Dockerfile`), a script for creating images.
- **Docker compose**, (`docker-compose.yml`), is a tool for defining and running multi-container Docker applications.
- **Docker Swarm**, is a clustering and scheduling tool for Docker containers.

A container packages the application service or function with all of the libraries, configuration files, dependencies and other necessary parts to operate.

# Dockerfile BUILD Commands

- `FROM` ‚Äì The image the new image will be based on.
- `MAINTAINER` ‚Äì Name and email of the maintainer of this image.
- `ARG` ‚Äì Environment variable in the container only for build stage.
    - Can be overridden with command line parameters.
- `COPY` ‚Äì Copy a file or a directory into the image.
- `ADD` ‚Äì Same as `COPY`, but handle URL:s and unpack tarballs automatically.
- `RUN` ‚Äì Run a command inside the container, such as apt-get install.
- `.dockerignore` ‚Äì Not a command, but it controls what files are added to the build context. Files not needed when building the image.
- `ONBUILD` ‚Äì Run commands when building an inherited Dockerfile.

---

# Dockerfile RUN commands

- `ENV` ‚Äì Set environment variable in the container.
- `CMD` ‚Äì Default command to run when running the container.
    - Can be overridden with command line parameters.
- `EXPOSE` ‚Äì Expose ports from the container.
    - Explicitly by *run* with `-p` or `-P`.
- `VOLUME` ‚Äì Specify that a directory should be stored outside the union file system.
    - Set by *run* with `-v`, default `/var/lib/docker/volumes`.
- `ENTRYPOINT` ‚Äì Specify a command that is not overridden as `CMD`. Give a default executable and use commands as parameters to it.
- `USER` ‚Äì Set the user for `RUN`, `CMD` and `ENTRYPOINT`.
- `WORKDIR` ‚Äì Sets the working directory for `RUN`, `CMD`, `ENTRYPOINT`, `ADD` and `COPY`.

# RATS - Creating the image <br/> Dockerfile (I)

```Dockerfile
FROM ubuntu:18.04

ENV HTTP_PROXY http://proxy.com:80/
ENV HTTPS_PROXY http://proxy.com:80/
ENV http_proxy http://proxy.com:80/
ENV https_proxy http://proxy.com:80/
```
